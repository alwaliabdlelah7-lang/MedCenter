# 🚀 نظام الأتمتة - MedCenter HIS Automation

تم إعداد أتمتة كاملة للمشروع عبر **GitHub Actions** و **Dependabot** لضمان تحديث المكتبات، البناء، والنشر التلقائي.

## 1. التحديث التلقائي للمكتبات (Dependabot)
- تم تفعيل **Dependabot** ليقوم بفحص التحديثات الجديدة لجميع المكاتب (npm) وملفات Gradle للموبايل أسبوعياً.
- سيقوم النظام بفتح **Pull Request** تلقائي بكل تحديث جديد، وما عليك سوى الموافقة لدمج التعديلات.

## 2. البناء والنشر التلقائي (CI/CD)
تم إعداد 3 مسارات عمل (Workflows):

### أ- البناء والتحقق (Main Pipeline)
- يعمل عند كل عملية `push` على فرع `main`.
- يقوم بالتأكد من خلو الكود من الأخطاء وبناء نسخة الويب للتجربة.

### ب- بناء النسخ النهائية والحلول (Release)
- عند إضافة **Tag** يبدأ بـ `v` (مثل `v1.0.0`)، سيقوم النظام تلقائياً بـ:
  - بناء تطبيق الأندرويد (APK).
  - بناء تطبيق الويندوز (Portable).
  - بناء تطبيق اللينكس.
  - إنشاء **GitHub Release** ورفع جميع الملفات إليه تلقائياً كـ (Assets).

### ج- النشر على Firebase
- يقوم بنشر نسخة الويب فوراً على Firebase Hosting عند التعديل على الكود الأساسي.

## 3. المتطلبات لتفعيل الأتمتة في حسابك
يجب عليك إضافة القيم التالية في إعدادات مستودعك على GitHub (**Settings > Secrets and variables > Actions**):

| Secret Name | Description |
| :--- | :--- |
| `FIREBASE_SERVICE_ACCOUNT` | مفتاح حساب الخدمة من Firebase |
| `FIREBASE_PROJECT_ID` | معرّف المشروع الخاص بك |
| `GEMINI_API_KEY` | مفتاح API الخاص بـ Google Gemini |
| `VITE_FIREBASE_CONFIG` | (اختياري) إعدادات الاتصال بقاعدة البيانات |
