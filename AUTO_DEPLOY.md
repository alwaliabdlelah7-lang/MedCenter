# 🚀 نظام الأتمتة الموحد - MedCenter HIS Automation

تم إعداد سحابة أتمتة شاملة وموحدة للمشروع عبر **GitHub Actions** لضمان تحديث الأمان، والبناء التلقائي، والنشر المجاني والكامل عبر جميع المنصات في غضون دقائق معدودة بمجرد رفع الكود!

---

## 📅 1. البناء المتكامل لكل المنصات (Unified Pipeline)

تم دمج كافة العمليات المشتتة سابقاً في خطة تدفق موحدة متكاملة وقوية تسمى **Unified Production Build & Auto-Deploy**:
- **نظام التشغيل المدعوم:** يتم تفعيل البناء متوازي العمليات (Parallel Build Matrix) على خوادم Linux وخوادم Windows من قِبل GitHub مجاناً.
- **التفعيل التلقائي:** يتم إطلاق عملية البناء الشامل لكل نظام بمجرد إجراء `push` للتعديلات على فرع `main` أو `master`.

---

## 🌐 2. منصات التصدير والنشر وكيفية الحصول عليها:

### 📱 أ- تطبيق الهواتف الذكية أندرويد (Android Mobile App)
- **ملف المخرجات:** ملف تثبيت مباشر بصيغة `MedCenter-HIS-v1.0.X.apk`.
- **طريقة عمل البناء:** يقوم خادم الأتمتة بتهيئة منفذ Java JDK وتكامل ملفات Gradle، ومن ثَم عمل مزامنة ذكية وحزم لمكونات الويب من قِبل Capacitor داخل بنية الأندرويد الأصلية وتصدير ملف تثبيت خالي من المشاكل.

### 💻 ب- تطبيق الكمبيوتر لويندوز (Windows Desktop App)
- **ملف المخرجات:** برنامج محمول مباشر العمل دون الحاجة للتثبيت بصيغة الـ Portable `MedCenter-HIS-v1.0.X.exe`.
- **طريقة عمل البناء:** يقوم الخادم السحابي للنظام (Windows Runner) باستدعاء برنامج Electron Builder لتغليف شفرة React وتجهيز ملف تشغيلي مباشر يعمل بسلاسة فائقة وسرعة قصوى مع تكامل واجهة المستخدم والوصول للعتاد المحلي بجودة كاملة.

### 🌐 ج- موقع الويب المباشر (GitHub Pages Live Web App)
- **الرابط والمستضيف:** مستضاف ومحمي بالكامل بشكل مجاني 100% على خوادم **GitHub Pages** الرسمية.
- **إعادة التحميل والمثالية:** يتضمن ملف `404.html` ذكي ومخصص لمنع كسر الروابط عند إعادة التحميل داخل الصفحات الفرعية للوحة المراقبة الخاصة بالنظام (React Router Route deep-linking support).

## 🎁 3. نظام الإصدارات المتكاملة (Integrated GitHub Releases)

بدلاً من الدخول لملفات البناء المؤقتة التي تنتهي صلاحيتها، يقوم النظام السحابي فور انتهاء البناء **بإنشاء إصدار رسمي تلقائي ومحدث (GitHub Release)** تحت اسم:
`الإصدار v1.0.X (تطبيقات المنظومة المتكاملة)`

يحتوي هذا الإصدار على:
1. تطبيق الأندرويد جاهز للتحميل والفتح على هواتف المستخدم والموظفين.
2. تطبيق الويندوز جاهز لتوزيع الهواتف المحمولة وأجهزة استقبال المشافي.
3. تفاصيل التحديث ورسائل التعديل الأخير للتسهيل على المطورين.

---

## 🛠️ 4. المتطلبات والخطوات لتفعيل الأتمتة في حسابك (مجاناً بالكامل)

### الخطوة الأولى: تفعيل روابط الويب (GitHub Pages)
1. بعد رفع الكود لمستودعك على GitHub، توجه إلى تبويب **Settings** ثم **Pages**.
2. تحت قسم **Build and deployment > Source**، حدد خيار **Deploy from a branch**.
3. تحت قسم **Branch**، اضغط القائمة المنسدلة واختر منها فرع `gh-pages` واجعل المجلد `/ (root)` ثم اضغط **Save**.
4. مبروك! سيقوم GitHub باستضافة موقعك الطبي فوراً وسيظهر رابط موقعك في أعلى الصفحة.

### الخطوة الثانية: تفعيل أذونات البناء والإطلاق السحابي
تأكد من السماح للـ Workflow بإنشاء الإصدارات عبر:
1. اذهب لمستودعك ثم **Settings > Actions > General**.
2. ابحث عن قسم **Workflow permissions** أسفل الصفحة للتحكم بالأذونات.
3. تأكد من تفعيل خيار **Read and write permissions** ثم اضغط **Save**.
*(هذا الإذن أساسي لكي يتمكن خادم البناء من رفع التطبيق النهائي APK و Executable إلى تبويب Releases الخاص بك).*
