# بيان النظام الشامل - MedCenter HIS (إصدار 2024 الكامل)

هذا المستند هو المرجع الرسمي والكامل لجميع مكونات نظام MedCenter HIS، بما في ذلك هيكل البيانات، الوظائف، الصلاحيات، وطرق النشر.

## 1. الهيكل الفني للنظام (Architectural Overview)
*   **الواجهة الأمامية (Frontend)**: React 18 + Vite + Tailwind CSS.
*   **الواجهة الخلفية (Backend)**: Express.ts (Node.js) متكامل مع Vite.
*   **قاعدة البيانات (Database)**: Google Firebase Cloud Firestore (NoSQL).
*   **الاتصال بالوقت الحقيقي**: Socket.io للتنبيهات والدردشة.
*   **النظام المتعدد المنصات**: 
    *   **Android**: عبر Capacitor.
    *   **Windows**: عبر Electron.
    *   **Web**: تطبيق ويب تدريجي (PWA).

---

## 2. هيكل البيانات (Database Schema - Firestore)

تم تصميم قاعدة البيانات لتشمل جميع جوانب العمل الطبي الإداري والسريري:

### أ. البيانات الأساسية (Core Records)
*   **المرضى (`patients`)**: رقم الملف، الاسم، تاريخ الميلاد، فصيلة الدم، الحساسية، التاريخ الطبي.
*   **الأطباء (`doctors`)**: التخصص، القسم، الهاتف، نسبة الطبيب، جدول المواعيد.
*   **طاقم التمريض (`nurses`)**: الاسم، القسم، وردية العمل.
*   **الأقسام (`departments`)**: (جراحة، باطنية، طوارئ، أطفال، إلخ).

### ب. العمليات السريرية (Clinical Operations)
*   **المواعيد (`appointments`)**: المريض، الطبيب، الوقت، الحالة (مؤكد، ملغي، تم).
*   **التنويم (`inpatients`)**: الغرفة، السرير، تاريخ الدخول، التشخيص، الطبيب المسؤول.
*   **العمليات الجراحية (`operations`)**: نوع العملية، الطاقم الجراحي، التخدير، تقرير العملية.
*   **المختبر (`lab_tests`)**: الطلبات، العينات، النتائج، المرجعية.
*   **الأشعة (`radiology`)**: النوع، التقرير، رابط الصورة.

### ج. الصيدلية والمخازن (Pharmacy & Inventory)
*   **الأصناف الدوائية (`pharmacy_items`)**: الاسم العلمي، التجاري، الكمية، تاريخ الانتهاء، السعر.
*   **الوصفات الطبية (`prescriptions`)**: الأدوية المرتبطة بزيارة المريض.

### د. النظام المالي (Financials)
*   **الفواتير/الإيصالات (`receipts`)**: الخدمات المقدمة، المبالغ، طريقة الدفع (نقدي، تأمين، آجل).

## 3. الوظائف والمميزات (Core Features)

1.  **لوحة تحكم ذكية**: عرض إحصائيات فورية للمرضى، الدخل، ونسبة الإشغال.
2.  **إدارة المواعيد**: حجز، تعديل، ومتابعة المواعيد وسجل الانتظار.
3.  **سجل المريض الإلكتروني (EMR)**: ملف متكامل لكل مريض يشمل جميع زياراته وفحوصاته.
4.  **نظام التمريض**: تسجيل المؤشرات الحيوية (Vital Signs) والعلاجات التمريضية.
5.  **الصيدلية الذكية**: تنبيهات عند اقتراب انتهاء الصلاحية أو نقص المخزون.
6.  **إدارة الحسابات**: نظام فوترة متكامل يدعم شركات التأمين والخصومات.
7.  **الدردشة الداخلية**: تواصل فوري بين الطاقم الطبي والإداري عبر Socket.io.

## 4. نظام الصلاحيات والأمان (Permissions & Security)

يتم التحكم بالوصول عبر **Firestore Security Rules** و **Middleware** في السيرفر:

*   **مدير النظام (Admin)**: صلاحيات كاملة (قراءة، كتابة، حذف، إدارة المستخدمين، تهيئة النظام).
*   **الطبيب (Doctor)**: قراءة السجلات الطبية، كتابة الوصفات، طلب فحوصات، عرض مواعيده الخاصة.
*   **الممرض (Nurse)**: تسجيل العلامات الحيوية، قراءة التعليمات الطبية، إدارة طلبات التمريض.
*   **موظف الاستقبال (Receptionist)**: إدارة المرضى، حجز المواعيد، إصدار الفواتير.
*   **الصيدلي (Pharmacist)**: صرف الأدوية، إدارة المخزون الدوائي.

*(ملاحظة: يتم التحقق من الصلاحيات عبر البريد الإلكتروني الموثق والتوثيق من طرف السيرفر).*

## 5. استيراد البيانات (Data Import Logic)

تمت إضافة وظيفة استيراد تلقائية لمساعدتك في رفع ملفات JSON الخاصة بك:

*   **نقطة الوصول (Endpoint)**: `POST /api/admin/import/:collection`
*   **طريقة الاستخدام**: أرسل مصفوفة (Array) من الأجسام (Objects) في جسم الطلب (Body).
*   **مثال**: لرفع 100 مريض دفعة واحدة، ارسل ملف JSON إلى `/api/admin/import/patients`.

## 6. البناء والنشر التلقائي (DevOps)

المشروع مهيأ تماماً للنشر التلقائي عبر **GitHub Actions**:

*   **بناء ويندوز**: يتم إنتاج ملف `Project Setup.exe` عبر Electron.
