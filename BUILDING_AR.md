# دليل البناء والنشر التلقائي - MedCenter HIS

تم إعداد هذا المشروع ليكون متكاملاً وقابلاً للبناء والنشر تلقائياً عبر GitHub Actions لجميع المنصات:

## 1. المنصات المدعومة
*   **Web (الويب)**: يتم بناء نسخة الويب وتحميلها كـ Artifact.
*   **Android (أندرويد)**: يتم بناء ملف APK تلقائياً.
*   **Tablet (اللوحي)**: مدعوم من خلال نسخة الأندرويد ونسخة الويب سريعة الاستجابة.
*   **Windows (ويندوز)**: يتم استخدام Electron لبناء ملفات تنفيد (.exe) قابلة للتثبيت.

## 2. الإعداد التلقائي لقاعدة البيانات (Firebase)
للاتصال التلقائي بقاعدة البيانات، يجب إضافة الأسرار (Secrets) التالية في مستودع GitHub الخاص بك:
*   `FIREBASE_SERVICE_ACCOUNT`: نص JSON الخاص بحساب الخدمة (Service Account).
*   `VITE_FIREBASE_API_KEY`: مفتاح الـ API للعميل.
*   (وبقية المتغيرات الموجودة في ملف `.env.example`).

## 3. كيفية التشغيل
عند القيام بـ `push` لأي كود جديد إلى فرع `main`:
1.  سيبدأ GitHub Actions ببناء نسخة الويب أولاً.
2.  بمجرد النجاح، سيبدأ بناء نسخ الأندرويد والويندوز في وقت واحد.
3.  ستجد الملفات الجاهزة (APK و EXE) في قسم **Actions** تحت مسمى **Artifacts**.

## 4. المكاتب المضافة
*   **Capacitor**: للتحويل إلى تطبيقات هواتف.
*   **Electron**: للتحويل إلى تطبيقات حاسوب (ويندوز).
*   **Firebase Admin**: للاتصال الآمن والمباشر بالسيرفر.
*   **Tailwind CSS**: لضمان التصميم المتجاوب مع جميع أحجام الشاشات.
